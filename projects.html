the "code breaker" Project

How to play:
"Code Breaker" is based on the board game "Master Mind". The secret code is made up of 4 colors, which can be red, green, blue, yellow. The player wins if he sets the right color in the right order. After every turn, the player gets feedback ...

  • how many of the set colors and positions where correct
  • how many colors where correct but the position was wrong

The score is calculated based on how many of the 10 lines are still free and how long the player needed to find out the secret code. If the play has not found out the secret code after the 10th turn, then he lost the game and does not get any points.

Features:

  • random secret code generation

  • 3 different high scores (daily, monthly, all time)
    (every high score list has a maximum of 20 entries, the lowest falls out)

  • the high score data is saved in a Google spreadsheet, every player writes to the same data base

  • the high scores get updated and can be viewed within the game

  • a live timer counting the time the player has played and points in real time

  • intuitive game and menu navigation (only arrow keys, some numbers and the '#' key are needed)

  • detailed feedback

  • sophisticated in game menu

the "weekend breaker" Project

What the project is about:
The website simulates an 'imaginary' travel agency. There is no "backend" and the website is static. The main goals are:

  • responsive layout across different screen sizes (mobile first design)

  • intuitive menu navigation & interactive elements

  • fast performance & error free site validation

Features:

  • navigation bar
    • intuitive icons
    • hover effect changes font color and shows hint
    • "hamburger" menu expanse when there is enough width
    • nav bar moves with the page down
    • unique design (buttons float over the page)

  • mobile first design
    • images are loaded by the browser according to the screen width
      (saves download time for mobile phones)
    • responsive layout
      ("product card" column changes from 1 to 3)
    • changing the browser font-size dose not break the design
      (media queries are in em and length units are in rem or %)
    • the images in the "product card" change every 10 seconds
      (to keep the user engaged)
    • user reviews change in intervals through slides
      (with the help of Bootstrap)
    • the price on the "booking" page uses CSS animation to create an flash effect
  • image sprite
    • three image sprites have been used to display together 24 images
      (increase performance by reducing the requests the browser needs to make to the server)
