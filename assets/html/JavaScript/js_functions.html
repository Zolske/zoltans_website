Arrow Functions

An arrow function is a compact alternative to a traditional function, but is limited and can't be used in all situations.

differences between arrow functions and traditional functions

  • Arrow functions don't have their own bindings to this or super, and should not be used as methods.
  • Arrow functions don't have access to the new.target keyword.
  • Arrow functions aren't suitable for call, apply and bind methods, which generally rely on establishing a scope.
  • Arrow functions cannot be used as constructors.
  • Arrow functions cannot use yield, within its body.

Syntax of an arrow function

The table shows six possible cases of arrow functions.
# pseudo code explanation
1. 
PARAMETER => EXPRESSION;
example

num => num * 2;
  • one parameter does not need parentheses
  • one expression does not need ...
    • curly brackets {}
    • a return statement (it is automatically applied)
2. 
(PARA_1, PARA_2) => {
    EXPRESSION_1;
    return EXPRESSION_2;
}
() => {
    EXPRESSION_1;
    return EXPRESSION_2;
}
example 
(num_1, num_2) => {
    let num_3 = num_2 - 4;
    return num_1 + num_3;
}
() => {
    let num_1 = 1 + 2;
    return num_1;
}
  • more or less than one parameter requirer parentheses
  • more than one expression do need ...
    • curly brackets {}
    • a return statement
3. PARAMETER => (OBJECT_LITERAL)
example

let message = textVar => ({foo: textVar});

message('hello');

// returning the object {foo: 'hello'}
  • To return an object literal expression
    requires parentheses around expression.
4. (a, b, ...r) => expression
  • Rest parameters are supported.
5. (a=400, b=20, c) => expression
  • Default parameters are supported.
6. ([a, b] = [10, 20]) => a + b; // result is 30
({ a, b } = { a: 10, b: 20 }) => a + b; // result is 30
  • Destructuring within parameters are supported.
