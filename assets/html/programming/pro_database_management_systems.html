Modeling data

It is better to plan the structure (schema) of the data model before creating it. Especially "SQL" databases are harder to change after. This can be done with the help of an " Entity-Relationship Diagram".

The general recommendation is to split the work into the following three design phases.

  1. Conceptual:
    the highest level phase where we map out the main entities, relationships, and attributes without a regard to storing them in the database yet.
  2. Logical:
    in this phase, we decide which entities and relationships will actually be stored in a relational database, and associate each with a table, containing attributes as columns.
  3. Physical:
    this is the lowest level phase in which we decide how exactly each of those tables should be stored in a computer.
